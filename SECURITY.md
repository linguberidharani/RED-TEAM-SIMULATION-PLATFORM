# Security Policy

Thank you for helping improve the security of the **AI-Assisted Red Team Simulation Platform**.

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest | ✅ Yes |
| Older Versions | ❌ No |

## Reporting a Vulnerability

Please **do not** report security vulnerabilities publicly.

Instead, contact:

**Maintainer:** Dharani Linguberi

**Email:** dharanipallaa@gmail.com

Please include:
- Description of the issue
- Steps to reproduce
- Impact
- Screenshots or proof of concept (if available)

## Responsible Disclosure

Please allow reasonable time to investigate and fix reported vulnerabilities before publicly disclosing them.

## Security Best Practices

- Never commit `.env` files.
- Never expose API keys or secrets.
- Keep dependencies updated.
- Validate and sanitize user input.
- Follow secure coding practices.

## Disclaimer

This project is intended **only for educational and research purposes**. It simulates cyberattack techniques in a safe environment and must not be used for unauthorized or illegal activities.
